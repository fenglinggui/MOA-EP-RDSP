# Product Instances

This folder contains the input product-instance data used in the formal EP-RDSP experiments.

## Files

- `product_instances.json`: complete definitions of the 15 problem instances.
- `product_instance_summary.json`: compact summary and field notes for the 15 problem instances.
- `ik_libraries/`: candidate inverse-kinematics libraries for the 15 product instances.
- `candidate_counts_by_product.json`: candidate count statistics by product.
- `ik_library_summary.json`: summary of the generated IK libraries.
- `ik_library_validation_summary.json`: validation summary for the generated IK libraries.

## Instance Fields

Each product contains `screws`, `tables`, and `dependencies`.

For screw operations:

- `position`: 3D operation coordinate `[x, y, z]`.
- `orientation`: operation normal/direction vector used by the robot target model.
- `screw_type`: screw specification label.
- `turns`: number of screw turns used by the unscrewing operation model.
- `dependencies`: explicit precedence constraints.
- `meta`: attributes such as `part_label`, `connect`, and group information.

For table/graspable-part operations:

- `position`: 3D part/operation coordinate `[x, y, z]`.
- `orientation`: removal/grasping normal or direction vector.
- `mass`: mass parameter of the graspable part.
- `d_grip`: grasping distance/diameter parameter.
- `dependencies`: explicit precedence constraints.
- `meta`: attributes such as `part_label`, `component`, `shape`, and notes.

The `dependencies` dictionary gives the final precedence constraints generated by `Product.build_dependencies()`, including explicit dependencies, screw-part connection constraints, two-table connection constraints, and conservative occlusion constraints.

The first five products are base instances. Products 6-15 are combined instances generated from the base products according to the component list in `product_instance_summary.json`.
