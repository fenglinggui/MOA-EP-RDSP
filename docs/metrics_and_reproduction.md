# Metrics and Reproduction Notes

This document describes the released input data and formal experimental results for the MOA-EP-RDSP dataset.

## Dataset Scope

The repository releases data and experimental results for the paper *Meta-operation-assisted optimization for execution-parameterized robotic disassembly sequence planning*.

The repository includes:

- product-instance definitions for 15 EP-RDSP instances;
- candidate IK libraries for the 15 product instances;
- RF training samples for MOA guidance;
- formal Experiment 1 and Experiment 2 summary results;
- lightweight per-run final Pareto fronts;
- paper-ready result figures.

Compared algorithm implementation code, full logs, intermediate populations, and large trained `.pkl` model files are not included.

## Input Data

### Product Instances

`input_data/product_instances/product_instances.json` contains the 15 problem instances. Each product includes:

- `screws`: unscrewing operations;
- `tables`: grasping/removal operations;
- `dependencies`: precedence constraints generated by the product model.

Key fields include:

- `position`: 3D coordinate `[x, y, z]`;
- `orientation`: operation normal/direction vector;
- `screw_type`: screw specification;
- `turns`: number of screw turns;
- `mass`: mass parameter of a graspable part;
- `d_grip`: grasping distance/diameter parameter;
- `meta.connect`: connection relation used to derive precedence constraints.

Products 1-5 are base instances. Products 6-15 are combined instances generated from the base products.

### RF Training Samples

`input_data/rf_training_samples/` contains the RF training rows used for MOA guidance:

- `grasp_training_rows.json`;
- `unscrew_training_rows.json`;
- `training_summary.json`.

The trained RF model files are not uploaded because they are large binary artifacts. The released JSON samples are intended to support data inspection and reproduction of the RF-guidance training process.

## Experiment 1

Experiment 1 compares MOA-assisted MOEAs with their corresponding base MOEAs under the same-HV setting.

Released methods:

- RVEA and MOA-RVEA;
- SMSEMOA and MOA-SMSEMOA;
- CTAEA and MOA-CTAEA;
- NSGAII and MOA-NSGAII.

Main settings:

- 15 product instances;
- 10 independent runs per method and product;
- population size 80;
- same-HV stopping criterion.

The released summary files are in:

`results/experiment1_moa_vs_moea_same_hv/summaries/`

The released per-run final Pareto fronts are in:

`results/experiment1_moa_vs_moea_same_hv/raw_pareto_fronts/`

The raw Pareto front file contains one record per method/product/run. Each record includes run metadata, final Pareto objective values, and the final Pareto solutions when available.

## Experiment 2

Experiment 2 compares MOA(ours) with other algorithmic frameworks under the same-time setting.

Released methods:

- CMOEMT;
- TriP;
- ARACMO;
- MOA(ours).

Main settings:

- 15 product instances;
- 10 independent runs per method and product;
- population size 80;
- same runtime budget per product group.

The released summary files are in:

`results/experiment2_moa_vs_frameworks_same_time/summaries/`

The released per-run final Pareto fronts are in:

`results/experiment2_moa_vs_frameworks_same_time/raw_pareto_fronts/`

## HV and IGD

Both objectives are minimization objectives:

- `F1`: execution time;
- `F2`: execution energy proxy.

The summary workbooks include the normalized HV and IGD values used in the formal result tables.

For Experiment 1, HV is computed using the HV-box and target settings released with the processed Experiment 1 summary.

## Statistical Test

Experiment 2 includes a 5% Wilcoxon signed-rank significance test in the released summary workbook.

The notation follows:

- `+`: MOA(ours) is significantly better than the baseline;
- `=`: no significant difference;
- `-`: MOA(ours) is significantly worse than the baseline.

## Figures

Paper-ready figures are released in `figures/`:

- `figures/background/`: background illustration for EP-RDSP;
- `figures/experiment1_hv_runtime/`: HV-runtime convergence figure for Experiment 1;
- `figures/experiment2_pareto_fronts/`: Pareto-front figure for Experiment 2;
- `figures/experiment2_statistics/`: statistical/boxplot figure for Experiment 2.

## Reproduction Notes

The released raw Pareto-front files are intended to support independent recalculation of HV, IGD, and statistical tests. Full execution logs and intermediate-generation populations are intentionally excluded to keep the repository compact.
